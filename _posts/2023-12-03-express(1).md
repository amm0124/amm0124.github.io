## express란?

express.js는 Node.js에서 인기있는 웹 프레임워크중 하나입니다.
기존의 HTTP 모듈을 사용해서 웹 서버를 만들 시, 읽기 힘들고, 확장성이 제한된다는 단점을 극복할 수 있습니다.

  

## express를 시작해보자

  

express를 사용하기 위해선, package.json이라는 file을 만들어야 합니다.
npm init을 터미널에 입력해서 쉽게 만들 수 있습니다.
코드를 수정하고, 서버를 재시작하는 번거로움을 nodemon 모듈을 사용하면 서버를 재시작하지 않아도 됩니다.
먼저 터미널에 들어가서, **npm init**을 입력합니다.
이를 통해서, package.json을 쉽게 만들 수 있습니다.


이후 **npm i express**를 통해서 express를 쉽게 설치할 수 있습니다.
npm i 모듈이름1 모듈이름2 모듈이름3 형식으로 여러 모듈을 한 번에 설치도 가능합니다.
예를 들어서 sequelize, mysql2, sequelize-cli 모듈을 설치하고 싶다면,
**npm i sequelize mysql1 sequelize-cli** <<를 터미널에 입력해서 빠르게 설치할 수 있습니다.

![npm i express 설치](경로)

정상적으로 설치가 되었다면 package.json에
dependencies 내부, express가 형성되었을 것입니다.

## express로 서버를 구동해보자

express로 서버를 구동하기 위해선 app.js라는 파일이 필요합니다.
C언어 계열로 치면, 이는 main.c와 유사합니다.
이 app.js에서 서버를 구동시킵니다.

    const  express  =  require('express');
	const  app  =  express(); //express 객체 생성.
	
	app.set('port', process.env.PORT  ||  3000); //3000번 포트를 사용합니다.
	
	// req : request 객체입니다. 요청에 대한 여러가지 정보를 담고 있습니다.
	// res : response 객체입니다. 응답에 대한 여러가지 정보를 담고 있습니다.
	// '/' 경로의 요청에 대해서 'Hello express!' 문자열을 보낸다는 의미입니다.
	// 문자열 경로의 요청에 대해서 응답을 하는 것을 라우팅(routing)이라고 합니다.
	app.get('/', (req,res) => {
		res.send('Hello express!');
	});

	//서버 요청 대기입니다. 3000번 포트에서 대기한다는 의미입니다.
	//만약 연결이 된다면 console에 3000번 포트에서 대기중입니다! 문자열을 출력합니다.
	app.listen(app.get('port'), ()=>{
		console.log(app.get('port'), '번 포트에서 대기중입니다!');
	}); 

이러한 app.js을 터미널에서 쉽게 실행시킬 수 있습니다.
package.json을 살짝 수정만 하면 되는데,

    {
		"name": "posting", //사용자 임의 지정
		"version": "1.0.0", //사용자 임의 지정
		"description": "express로 간단한 서버를 구축해보자", //사용자 임의 지정입니다. 
		"main": "app.js", //수정
		"scripts": {
			"test": "echo \"Error: no test specified\" && exit 1",
			"start" : "nodemon app"  //추가
		},
		"author": "",
		"license": "ISC",
		"dependencies": {
			"express": "^4.18.2"
		},
		"devDependencies": {
			"nodemon": "^3.0.2"
		}
	}

이렇게 수정을 하면 됩니다.
**package.json에서 주석은 존재하면 안됩니다!**

여기서 **dependencies**와 **devDependencies**의 차이가 궁금할 법도 합니다.
**dependencies**는 애플리케이션의 동작을 도와주는 라이브러리들이 저장되고, 
**devDependencies**는 개발 시 애플리케이션 실행을 도와주는 라이브러리들이 저장됩니다.
즉 애플리케이션 작동에는 **필요가 없는** 라이브러리들입니다. 
전자는 배포 시 필요하고, 후자는 배포 시 필요 없습니다.

이후 localhost:3000으로 접속을 시도합니다.
