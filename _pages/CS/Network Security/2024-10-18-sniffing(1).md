## Sniffing

`sniffing`의 사전적 의미는 코를 킁킁거리다라는 의미이다. 단어의 뜻에서 유추할 수 있듯이, 네트워크 상의 data를 도청 및 감청하는 공격을 의미한다. 이는 `Tempest`라는 전자기 신호 분석 장비를 사용하여 할 수도 있고, 실제로 케이블과 같은 물리적 네트워크 장비에 직접 접근하여 전기 신호를 분석할 수 있다.

## Promiscuous mode

혼잡 모드라고 한다. 일반적으로 수신한 프레임이 자신의 프레임이 아니면 기본으로 이를 버리는데, `Promiscuous mode`를 설정하면, 자신의 프레임이 아님에도 이를 버리지 않는다. 따라서 이 모드를 설정하면 모든 네트워크 트래픽을 수신 받을 수 있다. 이를 통해 패킷을 내부 데이터를 분석할 수 있다. Linux OS에선 

    ifconfig eth0 promisc
    
명령어를 터미널에 입력해서 Promiscuous mode로 변경할 수 있다. Window 기반에선 별도의 드라이버를 설치해야 한다.

## TCP Dump

`TCP Dump`는 원래 네트워크 관리를 하기 위한 도구였다. 이를 사용하면 인터페이스(eth0)내 송수신받는 패킷을 가로챌 수 있다. 이로 수집한 데이터는 법적 증거로 활용될 수 있다. 이는 수집된 데이터가 `raw data`임을 의미한다. 이는 기본으로 `Linux 환경`에서 사용되는데, 기본적이지만 매우 강력하다. 이를 통해 개인의 id나 password를 sniffing할 수 있다. 이제 과정을 알아보자. client는 Ubuntu desktop 14, server는 `Telnet`을 사용하고, Ubuntu 16 server를 사용한다. server의 IP 주소는 192.168.0.2다. `Telnet`는 원격지의 컴퓨터에 접속하여 명령어를 실행할 수 있는 네트워크 프로토콜중 하나이다. 텔넷은 데이터를 암호화하지 않고 평문으로 전달하기에, 보안 위험성이 있다.

먼저 client는 tcp dump를 설치해야 한다. 이후, tcp dump를 실행한다. -i eth0은 첫 번째 이더넷 네트워크 인터페이스를 캡처한다는 의미다. -xX는 16진수로 raw data를 parsing한다는 의미다. 

    sudo apt-get install tcpdump
    sudo tcpdump -i eth0 -xX host 192.168.0.2 

이후, 새롭게 터미널을 열고, client는 먼저 텔넷을 통해 원격 서버(192.168.0.2)에 연결 및 로그인을 해야 한다.
 
    telnet 192.168.0.2 
    id, password 입력
